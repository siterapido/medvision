# Glossary & Domain Concepts

## Core Concepts

### Agents
Specialized AI assistants designed for specific dental tasks.
- **Odonto GPT**: General-purpose dental assistant.
- **Research Agent**: Specializes in searching and summarizing dental literature.
- **Diagnostic Agent**: Analyzes images or descriptions to suggest diagnoses.

### Artifacts
Tangible outputs generated by the user or agents that are persisted in the database.
- **Research**: A saved search session with summaries and references.
- **Image Analysis**: An analysis report of a dental image.
- **Material**: Educational content or reference material.

### RAG (Retrieval-Augmented Generation)
The technique used by agents to provide accurate, grounded answers by retrieving relevant information from a knowledge base (vector store) before generating a response.

## Technical Terms

- **Agno**: The Python framework used for building the AI agents.
- **Supabase**: The backend-as-a-service platform providing the database, auth, and vector store.
- **Vector Store**: A specialized database (pgvector in Supabase) that stores embeddings of text to enable semantic search.
- **Embeddings**: Numerical representations of text used for semantic search.

## User Personas

- **Dentist/Professional**: The primary user who interacts with the agents to get assistance with clinical cases, research, or administrative tasks.
- **Admin**: A user with elevated privileges who manages the system (pipeline, users, etc.).

## Key Entities (Database)

- `users`: Registered users of the platform.
- `chats`: Conversation history.
- `messages`: Individual messages within a chat.
- `artifacts`: Generic table (or set of tables like `researches`, `image_artifacts`) for storing saved content.
